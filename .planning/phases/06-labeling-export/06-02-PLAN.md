   - `selectedDevice: string` — for save filename suggestion

2. **Template structure:**
   - Modal dialog wrapper (similar to ImportModal styling)
   - Title: "Export Data"
   - JSON display area:
     - `<textarea readonly>` or `<pre><code>` showing `JSON.stringify(entries, null, 2)`
     - Make text selectable (user can manually select + copy if clipboard API fails)
   - Button area:
     - "Copy to Clipboard" button
     - "Save to File" button
     - "Close" button
   - Status message area (optional: show "Copied!" briefly after copy)

3. **Computed:**
   - `jsonString: ComputedRef<string>` — `JSON.stringify(entries, null, 2)`

4. **Functions:**
   - `copyToClipboard()`:
     - Calls `navigator.clipboard.writeText(jsonString.value)`
     - On success, optionally show "Copied!" message for 2 seconds
     - On error, alert user or show error message
   - `saveToFile()`:
     - Check if `'showSaveFilePicker' in window` (feature detection)
     - If available (Chrome/Edge): use showSaveFilePicker (see below)
     - If not available (Safari): fall back to download via `<a download>`
   - `downloadFile()`:
     - Create a Blob from jsonString
     - Create object URL
     - Create `<a>` element with download attribute
     - Click to trigger download
     - Revoke URL and clean up

5. **File save implementation (showSaveFilePicker pattern):**
   ```typescript
   async function saveToFile() {
     try {
       const handle = await (window as any).showSaveFilePicker({
         suggestedName: `${selectedDevice.value || 'benchmark'}.json`,
         types: [{ description: 'JSON', accept: { 'application/json': ['.json'] } }]
       })
       const writable = await handle.createWritable()
       await writable.write(jsonString.value)
       await writable.close()
       // Optional: show success message
     } catch (err: any) {
       // If user cancels or API not supported, fall back to download
       if (err.name !== 'AbortError') {
         console.error('Save failed, falling back to download:', err)
       }
       downloadFile()
     }
   }
   ```

6. **Emit close event:**
   - `@click="$emit('close')"` on Close button

7. **Styling (scoped):**
   - Modal overlay with semi-transparent background
   - Modal dialog box with white background, padding, rounded corners
   - Textarea: full width, monospace font, height ~400px, dark text on light background
   - Buttons: standard button styling (blue primary, gray secondary)
   - Error/success messages in appropriate colors

**Notes:**
- Keep the modal simple and focused on display + actions
- Do NOT validate or modify the JSON; show it as-is
- Copy button should be accessible (not hidden)
- Save file suggestion should use `${selectedDevice.value}.json` pattern

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User export → file system | User exports JSON to disk; must use File System Access API (user confirms save location) |
| JSON data → clipboard | Copy to Clipboard uses navigator.clipboard API (browser sandboxed) |
| Edited entries → export JSON | mutableEntries contain user edits; exported JSON is final state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Data Leakage | Copy to Clipboard | accept | Clipboard API is browser-sandboxed; user explicitly clicks Copy; data stays in browser memory until export |
| T-06-05 | File Integrity | showSaveFilePicker | accept | File System Access API enforces user-initiated write (user must confirm save dialog); no silent overwrites |
| T-06-06 | Browser Compat | Safari fallback | mitigate | Detect showSaveFilePicker availability; fall back to download link for unsupported browsers |
| T-06-07 | User Confusion | Export button visibility | mitigate | Export button only appears when isDirty or isLabelingMode; clear visual indication of button state |
</threat_model>
