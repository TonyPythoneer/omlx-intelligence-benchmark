  <name>Task 1: Build Vue SPA locally and verify dist/ output</name>
  <files>.github/workflows/cd-static.yml</files>
  <action>
  Build the Vue SPA and verify the output is correct:
  
  1. In the project root, run: `pnpm install --frozen-lockfile` (ensure clean state)
  2. Build the Vue SPA: `pnpm exec vite build`
  3. Verify the build succeeds with no errors or warnings about critical issues
  4. Check that dist/ directory was created with expected structure:
     - `dist/index.html` exists (the built entry point)
     - `dist/assets/` exists with bundled JS, CSS, and any other assets
     - Check file sizes are reasonable (index.html > 1KB, assets bundle > 10KB)
  5. Verify vite.config.ts build configuration is correct:
     - `outDir: '../dist'` — build output goes to dist/ at root
     - `emptyOutDir: true` — dist/ is cleaned before each build
     - `publicDir: '../public'` — public/ files (data, settings.json) are copied to dist/
  6. Verify public/ symlinks are set up correctly (if using symlinks):
     - Check if public/data → app/data exists
     - Check if public/settings.json → app/settings.json exists
     - If symlinks don't exist, they should be created (or build would fail)
  7. If the build succeeds and dist/ looks correct, proceed to Task 2
  8. If build fails:
     - Check error messages (usually TypeScript, missing imports, config issues)
     - Verify vite.config.ts is correct (root: 'src', publicDir: '../public')
     - Ensure src/main.ts and src/App.vue are not broken
     - Consult Phase 1 plans for scaffold/build configuration
  </action>

## Threat Model

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions | CI/CD runs on GitHub servers; deploying to GitHub Pages (trusted host) |
| npm/pnpm registry | Dependencies installed in CI; frozen lockfile ensures reproducibility |
| dist/ build output | Generated from src/; should be deterministic and reproducible |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-06 | Tampering | cd-static.yml workflow | mitigate | Workflow is version controlled; changes reviewed before merge; build is deterministic |
| T-07-07 | Elevation | GitHub Actions build | accept | Build runs in isolated GitHub Actions container; no elevated privileges for deployment |
| T-07-08 | Accountability | Deployment | mitigate | GitHub Pages deployment logs are retained; build logs stored in Actions history |
