  If any CPs failed in Task 1, diagnose and fix the root causes:
  
  1. Examine the failure log: `cat outputs/ui-validation/final_runs/run_1/final_script_log.txt` to identify which CPs failed
  2. Review the error messages — common issues:
     - **Selector not found:** The Vue DOM structure differs from legacy app (e.g., ID changed, class changed, or element doesn't exist)
     - **Assertion failed:** The behavior doesn't match (e.g., count mismatch, text content different)
     - **Timeout:** Element took too long to appear or selector is wrong
  3. For each failing CP:
     a. **Identify the selector:** Find the failing line in final_script.py (e.g., `page.locator("#import-data")` for CP7)
     b. **Inspect the Vue app:** In the running app, open browser DevTools (F12), go to Elements, and search for the expected element
     c. **Determine if selector needs updating:** 
        - If element exists with different ID/class → update the selector
        - If element doesn't exist → verify Vue component should render it (check App.vue, component templates)
        - If behavior is different → trace the issue in Vue code (composables, computed properties, event handlers)
     d. **Update selector or code as needed:**
        - If selector: edit final_script.py with correct selector (e.g., `button:has-text('+ Import')` or `#import-btn`)

  4. Re-run Task 1: `python3 outputs/ui-validation/final_script.py`
  5. Repeat until all 11 CPs pass
