# Plan 10-01 — Dialog → reka-ui; Select retained native

## Goal
`ui/dialog.vue` rebuilt on reka-ui `Dialog*` (focus-trap, Escape, scroll-lock, aria) with its public API (`open`/`title`/`class` props, `close` emit, default + `footer` slots) preserved so `ImportModal`/`ExportModal` are unchanged. `ui/select.vue` retained as the native-`<select>` wrapper (documented decision). CLAUDE.md convention clarified.

## Tasks

### Task 1 — Rebuild ui/dialog.vue on reka-ui (UIPRIM-01)
- Replace Teleport/Transition/backdrop with `DialogRoot`(:open + @update:open→emit close) > `DialogPortal` > `DialogOverlay`(backdrop) + `DialogContent`(panel) > `DialogTitle` + X close button + default `<slot/>` + `#footer` slot.
- Preserve cva/`cn()` styling (rounded-xl, max-w-2xl default, border, shadow, backdrop blur). `class` prop still merges (Export overrides to max-w-3xl).
- Map `@update:open(false)` → `emit('close')`; keeps `@close` contract. Escape + outside-click + X all close.
- **Verify:** `grep -L 'reka-ui' src/components/ui/dialog.vue` empty (file imports reka-ui); no `<Teleport` left; `make test` green; headless: open Import + Export, close via X / Escape / outside-click each works; focus trapped.

### Task 2 — Retain native select + clarify convention (UIPRIM-02)
- Leave `ui/select.vue` native `<select>` as-is. Amend CLAUDE.md "UI component conventions": native HTML controls (`<select>`, `<input>`) are acceptable platform primitives; the reka-ui rule targets CUSTOM widgets that would otherwise be hand-rolled from divs.
- **Verify:** DeviceSelector still switches device (headless).

### Task 3 — Consumer regression (UIPRIM-03)
- Confirm ImportModal/ExportModal open, render, and close identically; Export footer buttons (copy/save) work; Import apply flow intact.
- **Verify:** `make test` green; Playwright baseline-vs-after zero new failures; screenshot of an open modal.

### Task 4 — checkpoint:human-verify (self-approved in autonomous mode)
Visual + interaction acceptance via headless evidence + screenshots.
