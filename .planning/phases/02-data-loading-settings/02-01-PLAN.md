Create src/composables/useSettings.ts that exports a useSettings() composable function. This composable fetches /settings.json and returns reactive state.

Define the following TypeScript interfaces:

```typescript
interface DeviceMeta {
  family: string;
  variant: string;
  memory: string;
  gpus: number;
}

interface Settings {
  defaultDevice: string;
  parametersBreakpoints: number[];
  devices: Record<string, DeviceMeta>;
}
```

Implement useSettings() to:
1. Use ref() and reactive() to create mutable state for the fetched settings
2. Use onMounted(() => { ... }) to fetch('/settings.json') when the component using this composable mounts
3. Parse the JSON response
4. Assign to settings ref
5. Return an object with:
   - `settings: Ref<Settings | null>` (null while loading or on error)
   - `defaultDevice: Ref<string | null>` (will be settings.defaultDevice once loaded)
   - `parametersBreakpoints: Ref<number[]>` (will be settings.parametersBreakpoints once loaded)
   - `devices: Ref<Record<string, DeviceMeta>>` (will be settings.devices once loaded)
   - `isLoading: Ref<boolean>` (true while fetch is in-flight, false when done)
   - `error: Ref<string | null>` (error message if fetch fails, null otherwise)

Error handling: if fetch fails, catch the error, store error.message in the error ref, and log to console. Do not re-throw.

Use standard Vue 3 Composition API (ref, reactive, onMounted, computed). Import these from 'vue'.

Guard against SSR: wrap the fetch call in `if (typeof window !== 'undefined')` to prevent errors in SSG context (though this is an SPA, defensive coding is good).

| Browser ↔ /data/*.json | Public JSON files, no authentication required; assumption is that benchmark data is public (confirmed: current app/index.html serves this data publicly) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | fetch('/settings.json') | accept | Settings contain only device metadata (family, variant, memory, gpus), no PII or secrets; publicly visible |
| T-02-02 | Denial of Service | fetch('/data/{device}.json') | accept | Large JSON files (potentially >1MB) could cause browser memory issues; mitigated by browser's built-in limits and user's ability to refresh |
| T-02-03 | Tampering | public/*.json files | accept | Static files served as-is; no runtime modification; CI data-validation ensures data integrity |
| T-02-SC | Tampering | npm install (if any new deps) | mitigate | No new external dependencies in this phase; only using Vue 3 + Vite already in place |
</threat_model>
