  <action>
Replace the stub applyImport() function in useImport.ts with complete implementation. Signature (already in place from Plan 1): `applyImport(currentEntries: Ref<Entry[]>): void`.

Implementation logic:
1. Create a Map for fast lookup: `const byModel = new Map(currentEntries.value.map((e, i) => [e.model, i]))`
2. For each parsedEntry in parsedEntries:
   - If byModel.has(parsedEntry.model) (OVERWRITE case):
     - Get index: `const idx = byModel.get(parsedEntry.model)`
     - Preserve all fields except scores: `currentEntries.value[idx] = { ...currentEntries.value[idx], scores: parsedEntry.scores }`
   - Else (NEW case):
     - Create full Entry object:
       - `model: parsedEntry.model`
       - `date: today's YYYY-MM-DD` (use isTodaysDate() helper)
       - `spec: { parameters_b: parsedEntry.spec.parameters_b, quantization: parsedEntry.spec.quantization, size_gb: parsedEntry.spec.size_gb }` (use values filled by user in modal)
       - `deprecated: false`
       - `tiers: { opus: false, sonnet: false, haiku: false }`
       - `abilities: { thinking: false, mtp: false }`
       - `scores: parsedEntry.scores`
     - Push to currentEntries.value: `currentEntries.value.push(newEntry)`
3. After merge, clear import state:
   - `importText.value = ''`
   - `isModalOpen.value = false`
4. Do NOT mutate parsedEntries or parsedEntry objects — create new Entry objects only

Per IMP-03: spec / abilities / tiers / deprecated are only set from the parsed values, not from form (form only affects NEW entries' spec field). OVERWRITE preserves all fields except scores. Type Entry correctly; no stub implementations.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Modal textarea paste → Parser → Merge | User-pasted stdout flows through parser to applyImport; merge writes to mutableEntries (in-memory only, no persistence risk) |
| Merged entries → Table rendering | New/updated entries are in mutableEntries; table re-renders via filteredEntries computed (read-only render) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Tampering (Merge logic) | useImport.applyImport() | mitigate | Merge creates new Entry objects; no direct mutation of input (except mutableEntries); OVERWRITE preserves all fields except scores (prevents accidental data loss) |
| T-05-05 | Denial of Service (Large merge) | applyImport() + table re-render | accept | Merge is O(n) Map-based; table re-render is O(m) where m = filteredEntries. Large imports (>1000 entries) may cause brief UI stall; observable via performance (user can see in browser devtools) |
| T-05-06 | Information Disclosure (In-memory entries) | mutableEntries ref | accept | Entries in memory only; no backend/network access; refresh clears data (designed behavior) |
</threat_model>
