<task type="auto">
  <name>Task 1: Create useImport composable with state and parsed-entry list</name>
  <files>src/composables/useImport.ts</files>
  <action>
Create `src/composables/useImport.ts` — a new composable that manages import modal state and entry parsing. Must export:

- `isModalOpen: Ref<boolean>` — controls modal visibility
- `importText: Ref<string>` — textarea content from user paste
- `parsedEntries: ComputedRef<ParsedResult[]>` — result of parsing importText via `parseImportInput()` from import.mjs. Each item has:
  - `status: 'NEW' | 'OVERWRITE'` — whether model already exists in currentEntries
  - `model: string` — model name
  - `scores: Scores` — parsed scores from stdout
  - `spec: { parameters_b, quantization, size_gb }` — for NEW entries, defaults { null, '', null }; for OVERWRITE, this is just the placeholder (user fills in NEW entries only)
  - `specFilled: boolean` — true only for NEW entries when all three spec fields are non-empty/non-null

- `isApplyEnabled: ComputedRef<boolean>` — true when importText is non-empty AND all NEW entries have specFilled === true

- `applyImport(currentEntries: Ref<Entry[]>): void` — merges parsedEntries into currentEntries (mutates in place):
  - For each parsedEntry, check if entry.model exists in currentEntries
  - If NEW: push a complete Entry with model, date (today's YYYY-MM-DD), spec (filled from form), abilities { thinking: false, mtp: false }, tiers { opus: false, sonnet: false, haiku: false }, deprecated: false, scores
  - If OVERWRITE: find existing entry by model, replace only the scores field; preserve spec / abilities / tiers / deprecated

- `openModal(): void` — set isModalOpen = true
- `closeModal(): void` — set isModalOpen = false, clear importText

Helper functions inside composable:
- `parseImportText(text: string): ParsedResult[]` — calls parseImportInput(text) from import.mjs, then enriches each result with { status, specFilled, spec }
- `isTodaysDate(): string` — returns today's date in YYYY-MM-DD format (use `new Date().toISOString().split('T')[0]`)

Per IMP-01, IMP-02, IMP-03, IMP-04. Type all refs and computed properties. No stub implementations.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Localhost guard → Hostname header | Import button hidden on non-localhost, but attacker could spoof window.location.hostname (mitigated by browser same-origin policy) |
| Textarea paste → Parser | User-pasted stdout is parsed; malformed input is ignored silently (no RCE risk, output is just Entry objects) |
| Parsed entries → In-memory state | Merged entries are added to mutableEntries; no persistence (in-memory only, disappears on refresh) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering (Hostname guard) | ImportModal button visibility | mitigate | Client-side check via window.location.hostname === 'localhost' or '127.0.0.1'; browser same-origin policy prevents spoofing |
| T-05-02 | Information Disclosure (Parsed data) | useImport.parsedEntries computed | accept | No PII in benchmark stdout; entries are technical metadata only; in-memory only (no persistence) |
| T-05-03 | Denial of Service (Large paste) | ImportModal textarea | mitigate | Textarea input length not constrained; parsing is O(n) in input length. Large pastes may slow browser. Observable via performance; user can close modal |
</threat_model>
