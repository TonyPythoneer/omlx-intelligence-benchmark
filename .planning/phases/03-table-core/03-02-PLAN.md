Add sorting capability to BenchmarkTable:

**State management:**
1. Add two local refs:
   - `sortCol: Ref<string>` initialized to 'date'
   - `sortDir: Ref<1 | -1>` initialized to -1 (DESC)

**Helper function (getSortValue):**
2. Create `getSortValue(entry: Entry, col: string): any` that extracts sortable values:
   - 'date': parse entry.date as `new Date(entry.date).getTime()` (sortable number)
   - 'spec.parameters_b': entry.spec.parameters_b (number, handle null)
   - 'spec.quantization': entry.spec.quantization (string)
   - 'spec.size_gb': entry.spec.size_gb (number, handle null)
   - 'scores.{BENCHMARK}.accuracy': nested path like 'scores.MMLU.accuracy' → extract value
   - 'scores.{BENCHMARK}.time_s': same pattern for time
   - Return null for missing/undefined values

**Computed sortedEntries:**
3. Create computed property `sortedEntries` that:
   - Returns a new sorted copy of entries (use [...entries].sort(...))
   - Sorts by current sortCol using getSortValue(entry, sortCol)
   - Applies sortDir: 1 = ASC, -1 = DESC (multiply comparison result by sortDir)
   - Null values always go to end (return 1 if only a is null, -1 if only b is null)
   - String comparisons: use `av.localeCompare(bv) * sortDir`
   - Number comparisons: use `(av - bv) * sortDir`

**Sort click handler (onSort):**
4. Create function `onSort(col: string)`:
   - If col === 'model' or th has 'no-sort' class, return early (do not sort)
   - If sortCol === col: toggle sortDir (1 ↔ -1)
   - Else: set sortCol = col and reset sortDir = 1 (ASC)
   - (sortedEntries recomputes automatically)

**Visual sort indicators:**
5. Add computed property `sortIndicator(col: string): 'asc' | 'desc' | ''`:
   - Return 'asc' if sortCol === col && sortDir === 1
   - Return 'desc' if sortCol === col && sortDir === -1
   - Return '' otherwise

6. In Row 3 (leaf-row) thead, each sortable th:
   - Add `:class="{ 'sort-asc': sortIndicator(col) === 'asc', 'sort-desc': sortIndicator(col) === 'desc' }"`
   - Add `@click="onSort(col)"` handler
   - Model column th: add `@click.prevent.stop` or just skip handler (no-sort class prevents click)

**CSS for sort indicators** (if not already present):
```
thead tr.leaf-row th.sort-asc::after  { content: ' ↑'; color: #2563eb; }
thead tr.leaf-row th.sort-desc::after { content: ' ↓'; color: #2563eb; }
thead tr.leaf-row th.sort-asc,
thead tr.leaf-row th.sort-desc { color: #2563eb; background: #eff6ff; }
```

**Template change:**
7. In tbody, change from `v-for="entry in entries"` to `v-for="entry in sortedEntries"`

Test sequence:

- Click "Params" header: sort by parameters_b ASC, ↑ indicator visible
- Click "Params" again: sort DESC, ↓ indicator visible
